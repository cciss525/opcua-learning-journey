## 一、数据质量码

### 1.1 为什么需要质量码

OPC UA 每个数值自带时间戳和质量码——工业协议独有的，HTTP/MQTT 都没有。

| 质量码 | 含义 | 处理方式 |
|--------|------|----------|
| Good (0) | 数据可靠 | 正常入库 |
| Uncertain | 传感器抖动/信号弱 | 入库但标记警告 |
| Bad | 传感器坏了/PLC离线 | 不入库，只记日志 |

不用质量码的话，传感器坏了还往数据库写错误数据，后端拿脏数据做分析全白费。

### 1.2 read_value() vs read_data_value()

```python
# 旧方式 — 只拿数值（不知道数据可不可靠）
val = await node.read_value()       # → 19.0

# 新方式 — 拿完整 OPC UA 数据包
dv = await node.read_data_value()   # → 值 + StatusCode + 时间戳
dv.Value          # 数值（Variant 包装）
dv.StatusCode     # 质量码对象
dv.StatusCode.is_good()   # True = 数据可靠
dv.SourceTimestamp # 数据产生时间
```

### 1.3 采集程序质量过滤逻辑

```
read_data_value()
    ↓
StatusCode.is_good()?  ──是──→ INSERT (quality=0)
    ↓ 否
Uncertain?  ──是──→ INSERT (打标记)
    ↓ 否
Bad → 不入库，logger.warning()
```

## 二、日志系统

### 2.1 为什么需要日志

采集程序跑在后台，窗口关了看不到 print。日志同时写文件和屏幕，事后能查。

### 2.2 实现方式

```python
import logging

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    handlers=[
        logging.FileHandler("logs/collector_20260529.log", encoding="utf-8"),
        logging.StreamHandler(),  # 屏幕也输出
    ],
)
```

### 2.3 日志级别

| 级别 | 用途 |
|------|------|
| logger.info() | 连接成功、写入条数 |
| logger.warning() | 质量码异常、不确定 |
| logger.error() | 连接失败、数据全 Bad |

日志文件在 `logs/` 目录下，按日期命名。采集程序跑一天，就是一个完整的操作记录文件。

## 四、采集程序：轮询 → 订阅

### 4.1 为什么改

轮询是"每 500ms 问一次"，值没变也问 —— 低效。
OPC UA 原生支持订阅——值变了自动推 —— 高效。

### 4.2 改造对比

| | 轮询 (旧) | 订阅 (新) |
|---|---|---|
| 方式 | `read_data_value()` | `subscribe_data_change()` |
| 触发 | 每 500ms，不管变没变 | 值变了才触发 |
| 写入频率 | 14条 × 2次/秒 = 28条/秒 | ~40条/秒，每个变化都抓到 |
| 依赖 | `asyncio.sleep(poll_interval)` | OPC UA 自动推送 |

### 4.3 关键代码

```python
class DataChangeHandler:
    """OPC UA 订阅回调：数据有变化才触发"""
    
    def datachange_notification(self, node, val, data):
        """值变了自动调用，不用轮询"""
        dv = data.monitored_item.Value
        if dv.StatusCode.is_good():
            # 写入 SQL Server
            cursor.execute("INSERT INTO opcua_data (...) VALUES (...)")

# 创建订阅
sub = await client.create_subscription(period=200, handler=handler)
for nid in nodes:
    node = client.get_node(nid)
    await sub.subscribe_data_change(node, ..., sampling_interval=100)

# 保持连接，等数据自己来
while True:
    await asyncio.sleep(10)
```

## 五、SQL 数据分析查询

```sql
-- 某个变量最近 10 分钟趋势
SELECT collected_at, value, unit
FROM opcua_data
WHERE var_name = N'转速' AND collected_at >= DATEADD(MINUTE, -10, GETDATE())
ORDER BY collected_at;

-- 今天每小时均值/最大/最小
SELECT 
    var_name,
    ROUND(AVG(value), 1) AS 均值,
    MAX(value) AS 最大值,
    MIN(value) AS 最小值,
    COUNT(*) AS 采集次数
FROM opcua_data
WHERE collected_at >= CAST(GETDATE() AS DATE)
GROUP BY var_name;

-- 质量码分布
SELECT 
    CASE quality WHEN 0 THEN 'Good' ELSE 'Other' END AS 状态,
    COUNT(*) AS 条数
FROM opcua_data
GROUP BY quality;
```

## 七、踩过的坑（今日新增）

9. **StatusCode 比较**: `StatusCode == ua.StatusCodes.Good` 永远 False——`StatusCodes.Good` 是 int，`StatusCode` 是对象。要用 `StatusCode.is_good()`。
10. **read_data_value() 返回值**: `dv.Value` 是 Variant 对象，不是裸数值；用 `dv.Value.Value` 或 `hasattr` 判断。
11. **start.bat 中文编码**: `chcp 65001` + 中文在 Windows cmd 中可能闪退，改用纯英文 ASCII。
12. **CORS 跨域**: 8081 页面 fetch 8080 的数据被浏览器拦截，在 config_editor 里加 `/api/live-data` 代理转发解决。
